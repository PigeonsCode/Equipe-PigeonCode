Sprint Backlog

Além de, juntamente com o Product Owner, selecionar os itens e definir uma meta, o Time de Desenvolvimento também cria um plano de como o que foi selecionado será desenvolvido. Esse plano é geralmente expresso por tarefas a serem realizadas durante o Sprint. O conjunto de itens selecionados e seu respectivo plano é chamado de Sprint Backlog e é geralmente representado na forma de um Quadro de Tarefas.

Segue abaixo uma demonstração de como montar uma tabela de Sprint Backlog:

ID User Story Tarefas Status
1 Como usuário, quero criar uma conta com e-mail e senha, para acessar o app.
  • Criar tela de cadastro;
  • Implementar validação de email/senha;
  • Criar API de cadastro no backend;
  • Salvar dados no banco de dados.
Feito
2 Como usuário, quero pesquisar livros por título ou autor, para encontrar o que desejo.
  • Criar tela de busca;
  • Implementar barra de pesquisa;
  • Criar API de busca no backend;
  • Exibir resultados em lista.
Fazendo
3 Como administrador, quero cadastrar novos livros no sistema, para manter o acervo atualizado.
  • Criar tela de cadastro de livro;
  • Implementar formulário (título, autor, capa, gênero);
  • Criar API de cadastro de livros;
  • Salvar informações no banco de dados.
